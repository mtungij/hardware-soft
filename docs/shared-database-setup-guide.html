Hardex Staff and Customer Shared Database Setup

Purpose: make staff.buildcore.site and customer.buildcore.site use one shared MySQL database so customer registration, staff approval, and customer login all work from the same records.

Problem We Fixed

The server had two Docker containers:

hardex-staff
hardex-customer

Each container had its own MySQL database:

hardex-staff    -> hardex_staff_db
hardex-customer -> hardex_customer_db

Because of that, a customer created or registered in one site could not be seen by the other site. Staff could not approve accounts created in the customer portal, and customers created in staff could not login in the customer portal.

Correct Architecture

Both Laravel apps should connect to one shared database:

hardex-staff      -> Laravel app
hardex-customer   -> Laravel app
hardex-mysql      -> shared MySQL database

Both apps now use:

DB_HOST=hardex-mysql
DB_DATABASE=hardex_db
DB_USERNAME=hardex_user

Commands Used

1. Backup Staff Database

docker exec hardex-staff sh -lc 'MYSQL_PWD="StrongPassword123@" mysqldump --no-tablespaces -h127.0.0.1 -uhardex_user hardex_staff_db' > /root/hardex_staff_db_backup.sql

Check the backup file:

ls -lh /root/hardex_staff_db_backup.sql

2. Shared MySQL Container

The shared MySQL container was already running:

docker ps -a | grep hardex-mysql

Expected result:

hardex-mysql   mysql:8.4   Up

3. Import Staff Backup Into Shared Database

cat /root/hardex_staff_db_backup.sql | docker exec -i hardex-mysql mysql -uhardex_user -p"StrongPassword123@" hardex_db

4. Update Staff .env

docker exec hardex-staff sh -lc 'cd /var/www/html/hardex-staff && sed -i "s/^ *DB_HOST=.*/DB_HOST=hardex-mysql/; s/^ *DB_PORT=.*/DB_PORT=3306/; s/^ *DB_DATABASE=.*/DB_DATABASE=hardex_db/; s/^ *DB_USERNAME=.*/DB_USERNAME=hardex_user/; s/^ *DB_PASSWORD=.*/DB_PASSWORD=StrongPassword123@/" .env'

5. Update Customer .env

docker exec hardex-customer sh -lc 'cd /var/www/html/hardex-customer && sed -i "s/^ *DB_HOST=.*/DB_HOST=hardex-mysql/; s/^ *DB_PORT=.*/DB_PORT=3306/; s/^ *DB_DATABASE=.*/DB_DATABASE=hardex_db/; s/^ *DB_USERNAME=.*/DB_USERNAME=hardex_user/; s/^ *DB_PASSWORD=.*/DB_PASSWORD=StrongPassword123@/" .env'

6. Clear Cache and Run Migrations

docker exec hardex-staff sh -lc 'cd /var/www/html/hardex-staff && php artisan optimize:clear && php artisan migrate --force'
docker exec hardex-customer sh -lc 'cd /var/www/html/hardex-customer && php artisan optimize:clear && php artisan migrate --force'

7. Confirm Both Apps Use Same Database

docker exec hardex-staff sh -lc 'cd /var/www/html/hardex-staff && php artisan tinker --execute="dump(config(\"database.connections.mysql.host\"), config(\"database.connections.mysql.database\"), config(\"database.connections.mysql.username\"));"'
docker exec hardex-customer sh -lc 'cd /var/www/html/hardex-customer && php artisan tinker --execute="dump(config(\"database.connections.mysql.host\"), config(\"database.connections.mysql.database\"), config(\"database.connections.mysql.username\"));"'

Both should show:

"hardex-mysql"
"hardex_db"
"hardex_user"

Expected Flow Now

  1. Customer registers at https://customer.buildcore.site/customer/register.
  2. The account is saved in hardex_db.customer_accounts with status pending.
  3. Super admin opens https://staff.buildcore.site/customer-accounts.
  4. Super admin sees the pending account and approves it.
  5. Customer logs in at https://customer.buildcore.site/customer/login.

Email Change Made

The staff customer creation email was changed. It now welcomes the customer in Swahili and gives instructions to open the customer portal and create their account, instead of sending a password that may not work when databases were separate.

Current status: both containers confirmed they are using hardex-mysql, hardex_db, and hardex_user.
Important: do not change one container back to 127.0.0.1 unless MySQL is intentionally running inside that same container. For this shared setup, both apps should keep DB_HOST=hardex-mysql.
